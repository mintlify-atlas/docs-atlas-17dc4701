---
title: 'c2scanner.sh'
description: 'Continuously monitors network socket state and alerts on new connections'
---

`c2scanner.sh` polls the system's socket table every 500ms using `ss -tupn` and diffs each snapshot against the previous one. When anything changes — a new connection, a new listener, or a process opening a socket — it prints the diff to stdout.

## Usage

```sh
./c2scanner.sh
```

No arguments required. Run in a terminal or redirect output to a log file:

```sh
./c2scanner.sh | tee /var/log/c2scanner.log
```

## The full script

```sh
#!/bin/sh

mkdir -p scans

FNAME="$(date +%s)"
FNAME_LAST="$(date +%s)ProMax"

while true
do
        FNAME="$(date +%s)"
        ss -tupn > "scans/$FNAME"

        DIFF=$(diff "scans/$FNAME" "scans/$FNAME_LAST")
        if [ "$DIFF" != "" ];
        then
                echo "====== NEW STUFF HAPPENING ======"
                echo "event at $(date)"
                diff "scans/$FNAME" "scans/$FNAME_LAST"
        fi

        FNAME_LAST=$FNAME

        sleep .5
done
```

## How it works

- Creates a `scans/` directory in the current working directory to store snapshots.
- Each snapshot is saved to a file named after the current Unix timestamp.
- `diff` compares the current snapshot against the previous one.
- If the diff is non-empty, the event is printed with a timestamp and the full diff.
- The loop sleeps 500ms between polls.

<Note>
All output goes to stdout. Run in a dedicated terminal, or pipe to `tee` so you get live output and a log file simultaneously.
</Note>

## What to look for

`ss -tupn` shows TCP, UDP, and Unix socket connections with process names and PIDs. Changes in the diff can indicate:

- A new outbound TCP connection from an unexpected process
- A new listening port opened after competition start
- A known service connecting to an unexpected remote address
- A process with no expected network activity opening a socket

## Example output

```
====== NEW STUFF HAPPENING ======
event at Mon Jan  1 00:00:00 UTC 2025
1a2
> tcp   ESTAB  0  0  10.0.0.5:54321  192.168.1.100:4444  users:(("nc",pid=1337,fd=3))
```

<Tip>
Use `c2scanner.sh` first to observe what normal network activity looks like before switching to `angryc2scanner.sh`. This prevents accidentally killing legitimate services.
</Tip>

## Disk usage

Each snapshot is a separate file in `scans/`. On an active system `ss` output can be several kilobytes per snapshot. Over a long competition, `scans/` will accumulate many files. Monitor disk usage or clean up old snapshots periodically:

```sh
find scans/ -mmin +60 -delete
```
