title: 'funcs/passwordHasher.sh'
description: 'Generates a SHA-512 password hash for use in harden.sh and the hostfile'
---

`passwordHasher.sh` generates a SHA-512 password hash in the `$6$` format used by `/etc/shadow`. Pass the plaintext password as the first argument and the script prints the hash to stdout.

## Script

```sh
#!/bin/sh
if [ "$#" -eq 0 ]; then
  printf "Pass password in first arg\n"
  exit 1
fi
salt=$(openssl rand -hex 16)
hashed_password=$(openssl passwd -6 -salt "$salt" "$1")
printf "$hashed_password"
```

The script uses `openssl rand -hex 16` to generate a random 16-byte hex salt, then passes it along with the password to `openssl passwd -6` to produce a SHA-512 crypt hash.

## Usage

Pass the password as the first argument:

```sh
./funcs/passwordHasher.sh 'MyStr0ngP@ssword'
```

If you run the script without arguments, it prints a usage message and exits:

```
Pass password in first arg
```

## Output format

The output is a single SHA-512 crypt string in the standard `/etc/shadow` format:

```
$6$a3f1c2e4b8d7f9a0$<hash>
```

- `$6$` — identifies the SHA-512 crypt algorithm
- The middle segment is the randomly generated salt
- The final segment is the derived hash

A real output looks like:

```
$6$4b2f9e1a3c7d8f0e$WvK2mX9pLqRtN1cZsYbGhUjDkIoFxEaVwTnMyPzSrQcAeBgHiJlOuv3456789012
```

<Note>
The salt is re-randomized every time you run the script. Two runs with the same password will produce different hashes, both of which are valid.
</Note>

## How the hash is used

The hash goes into the third column of the hostfile used by `harden.sh`:

```
hostname  192.168.1.10  $6$salt$hash
```

`harden.sh` reads this value and sets it as the password for system accounts using `usermod -p`.

<Tip>
Generate one hash per competition if all hosts share the same password, or one hash per host for better security. Because the salt is random, even identical passwords produce unique hashes.
</Tip>

<Warning>
Avoid passing the password directly on the command line in a shared environment — it will appear in shell history and in the process list (`ps aux`). In sensitive contexts, consider using a subshell or sourcing the password from a file.
</Warning>
