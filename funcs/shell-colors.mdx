`shellColors.sh` defines ANSI escape code variables for colored and styled terminal output. Source it at the top of any script that needs colored output.

## Script

```sh
RED="${C}[1;31m"
SED_RED="${C}[1;31m&${C}[0m"
GREEN="${C}[1;32m"
SED_GREEN="${C}[1;32m&${C}[0m"
YELLOW="${C}[1;33m"
SED_YELLOW="${C}[1;33m&${C}[0m"
RED_YELLOW="${C}[1;31;103m"
SED_RED_YELLOW="${C}[1;31;103m&${C}[0m"
BLUE="${C}[1;34m"
SED_BLUE="${C}[1;34m&${C}[0m"
ITALIC_BLUE="${C}[1;34m${C}[3m"
LIGHT_MAGENTA="${C}[1;95m"
SED_LIGHT_MAGENTA="${C}[1;95m&${C}[0m"
LIGHT_CYAN="${C}[1;96m"
SED_LIGHT_CYAN="${C}[1;96m&${C}[0m"
LG="${C}[1;37m" #LightGray
SED_LG="${C}[1;37m&${C}[0m"
DG="${C}[1;90m" #DarkGray
SED_DG="${C}[1;90m&${C}[0m"
NC="${C}[0m"
UNDERLINED="${C}[5m"
ITALIC="${C}[3m"
```

`$C` is the ANSI escape character (`\033`), defined in the sourcing script before `shellColors.sh` is sourced.

## Defined variables

### Color variables

| Variable | ANSI code | Description |
|---|---|---|
| `RED` | `\033[1;31m` | Bold red |
| `GREEN` | `\033[1;32m` | Bold green |
| `YELLOW` | `\033[1;33m` | Bold yellow |
| `RED_YELLOW` | `\033[1;31;103m` | Bold red text on bright yellow background |
| `BLUE` | `\033[1;34m` | Bold blue |
| `ITALIC_BLUE` | `\033[1;34m\033[3m` | Bold italic blue |
| `LIGHT_MAGENTA` | `\033[1;95m` | Bold light magenta |
| `LIGHT_CYAN` | `\033[1;96m` | Bold light cyan |
| `LG` | `\033[1;37m` | Light gray |
| `DG` | `\033[1;90m` | Dark gray |
| `NC` | `\033[0m` | Reset — clears all formatting |

### Style variables

| Variable | ANSI code | Description |
|---|---|---|
| `UNDERLINED` | `\033[5m` | Blinking/underlined (terminal-dependent) |
| `ITALIC` | `\033[3m` | Italic text |

### `sed` pattern variables

Each color also has a matching `SED_*` variable. These are designed for use with `sed` to colorize matched text inline, using `&` to refer back to the matched string:

| Variable | Pattern |
|---|---|
| `SED_RED` | `\033[1;31m&\033[0m` |
| `SED_GREEN` | `\033[1;32m&\033[0m` |
| `SED_YELLOW` | `\033[1;33m&\033[0m` |
| `SED_RED_YELLOW` | `\033[1;31;103m&\033[0m` |
| `SED_BLUE` | `\033[1;34m&\033[0m` |
| `SED_LIGHT_MAGENTA` | `\033[1;95m&\033[0m` |
| `SED_LIGHT_CYAN` | `\033[1;96m&\033[0m` |
| `SED_LG` | `\033[1;37m&\033[0m` |
| `SED_DG` | `\033[1;90m&\033[0m` |

## How to use in scripts

Define the escape character `C`, then source `shellColors.sh`:

```sh
C=$(printf '\033')
. ./funcs/shellColors.sh
```

Then use the variables with `printf`:

```sh
printf "%sStarting hardening...%s\n" "$GREEN" "$NC"
printf "%s[WARNING]%s Firewall rules not found.\n" "$YELLOW" "$NC"
printf "%s[ERROR]%s Could not detect OS.\n" "$RED" "$NC"
```

To colorize matching text in `sed` output:

```sh
some-command | sed "s/ERROR/$SED_RED/g"
```

<Note>
The escape character is defined using `printf '\033'` for POSIX compatibility. Using `echo -e` is not portable across all `sh` implementations. Always define `C` this way before sourcing `shellColors.sh`.
</Note>

<Note>
The color variables defined in `shellColors.sh` are also defined inline in `baseline/standard.sh` and `baseline/specific.sh`. This file is the centralized, standalone version. If you are adding a new script to the suite, source this file rather than duplicating the definitions.
</Note>
